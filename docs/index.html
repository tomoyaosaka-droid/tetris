1. システム概要

依存ライブラリなし・ビルドステップなしの単一ファイル (tetris.html) で動作するブラウザ Tetris。 HTML / CSS / JavaScript のみで構成され、Canvas 2D API を用いて描画する。

公開 URL: https://practice-day1-osaka.web.app  |  GitHub: tomoyaosaka-droid/tetris
graph TB subgraph Browser HTML[tetris.html] subgraph JavaScript Bag Piece Board Game render["render()"] Input["Input Handler"] end subgraph Canvas2D BC["board canvas"] HC["hold canvas"] NC["next canvas"] end end HTML --> JavaScript HTML --> Canvas2D Game --> Board Game --> Bag Game --> Piece render --> BC render --> HC render --> NC Input --> Game

各クラスの責務

4. ゲームループ

requestAnimationFrame を使ったフレームループ。毎フレーム game.update(dt) → render(game) を呼ぶ。

flowchart TD A([requestAnimationFrame]) --> B["dt = min(ts - last, 100ms)"] B --> C["game.update(dt)"] C --> D{state === running?} D -- No --> E[skip] D -- Yes --> F["flashTimer 更新"] F --> G["DAS 処理\n(左右入力の連続移動)"] G --> H["重力処理\n(gravTimer += dt)"] H --> I{gravTimer >= speed?} I -- Yes --> J["move(0,1)\n落下1マス"] J --> K["onGround 判定"] K --> L{on ground?} L -- Yes --> M["lockTimer += dt"] M --> N{lockTimer >= 500ms?} N -- Yes --> O["_lock()\nピース固定"] O --> P["clearLines()\nライン消去"] P --> Q["_score(n)"] Q --> R["_spawn()\n次ピース"] E --> S["render(game)"] R --> S N -- No --> S L -- No --> S I -- No --> S S --> A

stateDiagram-v2 [*] --> idle : 起動 idle --> running : SPACE / タップ running --> paused : P / Esc paused --> running : P / Esc / タップ running --> over : ブロックがスポーン不可 over --> running : SPACE / タップ (new Game)

6. SRS 回転システム

Super Rotation System (SRS) を実装。回転時に最大 5 つのウォールキックを順番に試み、 最初に有効な位置で回転を確定する。

flowchart LR A["rotate(dir)"] --> B["from = piece.rot\nto = (from+dir+4)%4"] B --> C["key = from>to\nkicks = KJ or KI"] C --> D["kick[0] を試みる"] D --> E{valid?} E -- Yes --> F["回転確定\nghostY 更新"] E -- No --> G{次の kick?} G -- Yes --> D G -- No --> H["回転失敗 (return false)"]

座標系注意: KJ / KI は スクリーン座標 (y-down)。 TetrisWiki の y-up 表記から y 成分を符号反転して定義している。

テーブル対象ピース
KJJ / L / S / Z / T (3×3 ピース)
KII ピース専用
(なし)O ピース — キックなし [[0,0]]

flowchart TD A["ライン消去 n 本"] --> B["基本点 = LINE_PTS[n] × level"] B --> C{n === 4 ?} C -- Yes --> D{btb フラグ ON?} D -- Yes --> E["pts × 1.5\n(Back-to-Back Tetris)"] D -- No --> F["btb = true"] C -- No --> G["btb = false"] E --> H F --> H G --> H H{combo > 0?} -- Yes --> I["pts += 50 × combo × level"] H -- No --> J I --> J["score += pts"]
消去ライン数名称基本点 (lv=1)
1Single100
2Double300
3Triple500
4Tetris800
—Hard drop2 pts/行
—Soft drop1 pt/行

レベルは Math.floor(lines / 10) + 1（最大 20）。重力速度は (0.8 - (lv-1)×0.007)^(lv-1) × 1000 ms。

8. 入力システム (DAS)

Delayed Auto Shift (DAS) を実装。キーを押し続けると初回遅延後に連続移動する。

sequenceDiagram participant User participant Input participant Game User->>Input: ArrowLeft keydown Input->>Game: move(-1,0) (即時1回) Input->>Game: dasL=true, dasLT=0 loop 毎フレーム (update内) Game->>Game: dasLT += dt alt dasLT > DAS_DELAY(167ms) Game->>Game: reps = floor((dasLT - 167) / 33) Game->>Game: move(-1,0) × reps end end User->>Input: ArrowLeft keyup Input->>Game: dasL=false
定数値説明
DAS_DELAY167 ms初回遅延（キー押下から連続移動開始まで）
DAS_RPT33 ms連続移動の間隔

9. レンダリング

render(game) はゲーム状態を読み取り Canvas に描画する純粋な関数（副作用なし）。

flowchart TD R["render(game)"] --> B1["board canvas をクリア"] B1 --> B2["グリッド線描画"] B2 --> B3["固定ブロック描画\n(board.g)"] B3 --> B4["ゴーストピース描画\n(透過+アウトライン)"] B4 --> B5["アクティブピース描画"] B5 --> H1["hold canvas 描画\n(canHold=false で暗く)"] H1 --> N1["next canvas 描画\n(上位3ピース)"] N1 --> S1["スコア/レベル/ライン\nDOM更新"] S1 --> F1["フラッシュメッセージ\n(TETRIS! など)"] F1 --> O1["オーバーレイ表示制御\n(idle/paused/over)"]
Canvas 要素内容サイズ
#boardメインボード・ゴースト・アクティブピースBS×COLS × BS×ROWS (動的)
#hold-cvホールドピースプレビュー88×78 px
#next-cvNext キュー 3 ピース88×264 px

ブロックサイズ BS はウィンドウリサイズ時に動的に再計算され、最大 30px・最小 14px に収まる。
